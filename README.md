# ResumeAutoApplier

Сервис автоматических откликов на вакансии с управлением через Telegram. Под каждого кандидата находит компании на hh.ru, пишет персональное сопроводительное письмо и отправляет отклик — письмом или через форму на сайте.

## Возможности
- Можно менеджментить сразу множество кандидатов параллельно. 
- Поиск компаний под нужную должность через hh.
- Персональное сопроводительное письмо под каждую вакансию.
- Автоматическое заполнение и отправка форм отклика.
- Режимы **Тест** (безопасная проверка: кидаем письма сами себе, а на сайтах не сабмитим форму) и **Боевой** (реальная отправка).
- Отчёт по откликам в боте для каждого кандидата отдельно.

## Что нужно
Установленный **Docker Desktop** — [тут](https://www.docker.com/products/docker-desktop/). Больше ничего ставить не требуется.

## Запуск

### Шаг 1. Получить ключи доступа

| Сервис | Где получить | Поля в `.env` |
|---|---|---|
| Telegram-бот | [@BotFather](https://t.me/BotFather) → команда `/newbot` | `TELEGRAM_BOT_TOKEN` |
| Google Gemini | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) | `GEMINI_API_KEY` |
| GigaChat | [Sber Developers](https://developers.sber.ru/portal/products/gigachat-api) → ключ авторизации | `GIGACHAT_CREDENTIALS` |
| hh.ru | [dev.hh.ru](https://dev.hh.ru) → создать приложение | `HH_CLIENT_ID`, `HH_CLIENT_SECRET` |
| Почта для отправки | пароль приложения (Яндекс или Gmail) | `EMAIL_ADDRESS`, `EMAIL_APP_PASSWORD` |

**Детали про почту:** 
| Почта | Где создать пароль приложения | `SMTP_HOST` | `SMTP_PORT` |
|---|---|---|---|
| **Яндекс** | Яндекс ID → «Безопасность» → «Пароли приложений» → создать пароль для почты (smtp). Плюс включить IMAP в настройках почты*. | `smtp.yandex.ru` | `465` |
| **Gmail** | Включить двухэтапную аутентификацию, затем Аккаунт Google → «Безопасность» → «Пароли приложений» → создать. | `smtp.gmail.com` | `465` |

>Полученный пароль в `EMAIL_APP_PASSWORD`, адрес ящика в `EMAIL_ADDRESS` (в тестовом режиме сюда будут приходить письма на проверку).

>*IMAP в настройках для яндекс почты тут: "Все настройки" -> "Почтовые программы" -> поставить галочку на строке с IMAP.

### Шаг 2. Вписать ключи в файл `.env`
1. Скопировать шаблон: файл `.env.example` → переименовать копию в `.env`.
2. Вписать в `.env` значения из шага 1 (рядом с соответствующими полями, после знака `=`).

### Шаг 3. Запустить
В папке проекта выполнить одну команду:
```bash
docker compose up -d --build
```
Первый проход долго билдит тяжёлый образ. По окончании зайти в тг и выбрать команду `/menu`.

## Как пользоваться ботом
Команды: `/menu` — меню, `/check` — проверка настроек, `/cancel` — отмена.

1. **Добавить кандидата** — загрузить резюме (PDF/DOCX) и указать желаемую должность.
2. **Запустить** — выбрать количество компаний и режим: **Тест** (письма приходят на свою почту, формы не отправляются) или **Боевой** (реальная отправка).
3. **Результаты** — отчёт по компаниям появится в боте.

## Управление
- Проверить настройки: команда `/check` в боте.
- Остановить: `docker compose down` (данные сохраняются).
- Запустить снова: `docker compose up -d`.
